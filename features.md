# Feature Architecture

The full feature vector is **797 dimensions** per post, assembled from four blocks.

Generated by `notebooks/feature_engineering.ipynb` and saved to `data/features/`.

## Overview

| Block | Columns | Dims | Description |
|-------|---------|------|-------------|
| `embeddings` | [0:768) | 768 | Nomic-Embed-Text semantic embedding of post text |
| `topic_ohe` | [768:777) | 9 | One-hot encoded topic label (A–I) |
| `toxic_ohe` | [777:782) | 5 | One-hot encoded toxicity level (0–4) |
| `numeric` | [782:797) | 15 | Standardized numeric features (see below) |

## Block 1: Text Embeddings (768 dims)

Each post's title and content are concatenated into a single string and embedded
using **Nomic-Embed-Text** via Ollama. The model produces a 768-dimensional
dense vector that captures the semantic meaning of the post.

- Posts are sent to the Ollama API at full length when possible.
- The ~0.2% of posts that exceed the model's 2048-token context window are
  truncated to the maximum character count that fits, determined by binary search.
- 96.9% of all text across the dataset is preserved in the embeddings.
- These features are **not** standardized (they are already roughly unit-norm).

## Block 2: Topic Label — One-Hot (9 dims)

The dataset assigns each post a `topic_label` from A through I.
Each label is one-hot encoded into a 9-dimensional binary vector.

| Column | Feature | Topic |
|--------|---------|-------|
| 768 | `topic_A` | Identity |
| 769 | `topic_B` | Technology |
| 770 | `topic_C` | Socializing |
| 771 | `topic_D` | Finance / Crypto |
| 772 | `topic_E` | Philosophy / Existential |
| 773 | `topic_F` | Meta / Platform |
| 774 | `topic_G` | Art / Creative |
| 775 | `topic_H` | News / Current Events |
| 776 | `topic_I` | Other / Miscellaneous |

## Block 3: Toxicity Level — One-Hot (5 dims)

The dataset provides a `toxic_level` score from 0 to 4 for each post.
Each level is one-hot encoded into a 5-dimensional binary vector.

| Column | Feature | Level |
|--------|---------|-------|
| 777 | `toxic_0` | Safe — no toxic content |
| 778 | `toxic_1` | Mildly toxic — slight edge or insensitivity |
| 779 | `toxic_2` | Moderately toxic — clearly offensive language |
| 780 | `toxic_3` | Highly toxic — aggressive or harmful content |
| 781 | `toxic_4` | Malicious — severe toxicity, threats, or hate |

## Block 4: Numeric Features (15 dims)

These features are **standardized** (zero mean, unit variance) using a
`StandardScaler` fit on the training set only, then applied to val/test.

### Text length features

| Column | Feature | Description |
|--------|---------|-------------|
| 782 | `combined_word_count` | Total number of whitespace-delimited words in the post title + content combined. Measures post length in a way that is roughly proportional to reading time. |
| 783 | `sentence_count` | Number of sentences in the combined title + content, split on `.`, `!`, and `?` followed by whitespace. Captures structural complexity — posts with more sentences tend to be more developed arguments or narratives. |
| 784 | `title_char_len` | Character length of the post title only. Short titles may be less descriptive; very long titles may indicate clickbait or detailed questions. |
| 785 | `content_char_len` | Character length of the post body (content field). Captures raw post length at a more granular level than word count, since it includes punctuation, whitespace, and code formatting. |
| 786 | `has_url` | Binary (0 or 1) indicating whether the post includes an attached URL. Posts with links may drive different engagement patterns (link sharing vs. text discussion). Only ~0.5% of posts have URLs. |

### Temporal features

| Column | Feature | Description |
|--------|---------|-------------|
| 787 | `hour_of_day` | Hour (0–23 UTC) when the post was created, extracted from the `created_at` timestamp. Captures time-of-day effects on engagement — posts during peak hours may receive more visibility. |
| 788 | `day_of_week` | Day of the week (0 = Monday through 6 = Sunday) when the post was created. Captures weekly engagement cycles — weekday vs. weekend posting patterns. |

### Sentiment features

| Column | Feature | Description |
|--------|---------|-------------|
| 789 | `vader_compound` | VADER sentiment compound score ranging from -1 (most negative) to +1 (most positive). This is a single normalized score computed by NLTK's VADER lexicon-based sentiment analyzer, which aggregates word-level sentiment intensities with rules for negation, punctuation emphasis, and conjunctions. It summarizes the overall emotional tone of the entire post. |
| 790 | `tb_polarity` | TextBlob polarity score ranging from -1 (negative) to +1 (positive). Computed by averaging word-level polarity values from a pattern-based lexicon. Unlike VADER, TextBlob uses a simpler rule set and tends to produce scores closer to zero for long or mixed-sentiment texts. Provides a second independent sentiment signal. |
| 791 | `tb_subjectivity` | TextBlob subjectivity score ranging from 0 (fully objective / factual) to 1 (fully subjective / opinionated). Measures how much the post expresses personal opinions, feelings, or judgments vs. stating facts. Highly subjective posts may drive more engagement through agreement/disagreement dynamics. |

### Part-of-speech (POS) counts

Computed using NLTK's `pos_tag` on tokenized post text (title + content). Each count
represents the total number of words in the post that belong to that grammatical category.
These capture the linguistic composition of a post — e.g., noun-heavy posts tend to be
informational, verb-heavy posts tend to be action-oriented or narrative, and
pronoun-heavy posts tend to be personal or conversational.

| Column | Feature | Penn Treebank tags included | Description |
|--------|---------|---------------------------|-------------|
| 792 | `noun_count` | NN, NNS, NNP, NNPS | Count of all nouns — common nouns (singular and plural) and proper nouns. High noun counts indicate information-dense, descriptive, or entity-rich posts. |
| 793 | `pronoun_count` | PRP, PRP\$, WP, WP\$ | Count of all pronouns — personal (I, you, he), possessive (my, your), and interrogative (who, what). High pronoun counts suggest conversational, personal, or question-oriented posts. |
| 794 | `verb_count` | VB, VBD, VBG, VBN, VBP, VBZ | Count of all verb forms — base, past tense, gerund, past participle, present (non-3rd person and 3rd person singular). High verb counts indicate action-oriented or narrative posts. |
| 795 | `adjective_count` | JJ, JJR, JJS | Count of all adjectives — base, comparative (better), and superlative (best). High adjective counts suggest opinionated, descriptive, or evaluative writing. |
| 796 | `adverb_count` | RB, RBR, RBS | Count of all adverbs — base (quickly), comparative (faster), and superlative (fastest). Adverbs modify verbs and adjectives, adding nuance to how actions or qualities are described. |

## Target Labels

The target variable `engagement_class` is an integer 0–3 mapped from engagement score quartiles:

| Label | Class | Definition |
|-------|-------|------------|
| 0 | `lower` | Engagement score ≤ 0 (no upvotes, no comments, possibly only downvotes) |
| 1 | `lower_middle` | Bottom third of posts with positive engagement scores |
| 2 | `upper_middle` | Middle third of posts with positive engagement scores |
| 3 | `upper` | Top third of posts with positive engagement scores |

The engagement score is computed as:

```
engagement_score = log(1 + upvotes) + log(1 + comment_count) - 0.5 * log(1 + downvotes)
```

## Data Splits

| Split | Rows | Proportion |
|-------|------|------------|
| Train | 31,063 | 70% |
| Val | 6,656 | 15% |
| Test | 6,657 | 15% |

All splits are stratified on `engagement_class` to maintain balanced class proportions.
The `StandardScaler` for numeric features is fit on the training set only.
